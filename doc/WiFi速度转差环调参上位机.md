# Wi-Fi 速度环与转差环调参上位机

运行独立工具：

```powershell
.\.venv\Scripts\python.exe tool\speed_turn_tuner.py
```

依赖仅为现有 `tool/requirements.txt` 中的 PyQt5；没有相机标定依赖。电脑连接主板热点 `BALANCECAR_B10` 后，在工具中使用默认地址：主板 `192.168.4.1`、本地遥测 UDP `9000`、主板命令 UDP `9001`。

## 开始前的固件前置条件

本工作区固件已具备整定的必要前置修正：启动目标速度为 `0 m/s`；平衡内环按 `error × dt` 积分并在输出饱和时抑制积分累积；速度环同样具有抗积分饱和；转差环会根据混控器上一次实际施加的转向输出抑制积分累积；启动自检会进行静止陀螺仪零偏校准。上电后的约一秒校准期间必须让小车保持静止；校准失败不会通过自检。

固件遥测已升级为 `T,10`：保留转差请求输出和混控后实际输出，并追加三环饱和标志、编码器初始化有效性、IMU 校准状态以及平衡/速度控制周期。工具会显示并写入会话 CSV。整车页的“已确认”勾选仍是人工版本确认：只有确认已烧录该固件后，才会解锁所有非零 `Ki` 下发。

## 通信与安全边界

工具每秒发送一次 `H` 到 UDP 9001，主板随后将遥测单播至本机 UDP 9000。命令与现有主板固件一致：

```text
C,<seq>,DRIVE,<-0.250..0.250>
C,<seq>,TURN,<-0.200..0.200>
C,<seq>,TRACK,0|1
P,<seq>,speed,kp|ki|max_pitch,<value>
P,<seq>,turn,kp|ki|max,<value>
```

每个命令等待 `A,<seq>,...` ACK；未收到 ACK 时每 0.5 秒重发，最多三次。上位机不参与 50 Hz 控制闭环，姿态安全、限幅、滤波、积分与急停仍由主板执行。调参前让车轮悬空；地面测试时旁边必须有人可以立即点击“停止平衡”。

连接栏中的“开启摄像头循迹”仅在主板状态为 `BALANCING` 时可点击。它发送 `TRACK,1`，主板随后将相机 I²C 给出的目标左右轮速差接入转差环；按钮文字由 `T,10` 的 `visionTrackingEnabled` 回读更新。手动下发 `TURN` 会按主板既有安全逻辑自动关闭摄像头循迹。

## 曲线与指标

工具有三个控制器页面和一个整车整定页：

- **平衡角环整定**：实时显示俯仰角、角速度、角度误差、P/I/D 项及电机输出；在线调整 `balance.kp`、`balance.kd`、平衡点 `trim` 和归一化最大电机输出 `max_motor`。主板默认使用 PD；`balance.ki` 作为高级选项，只有在机械平衡点已确认后才应小步加入。
- **转速环调节**：在线调整 `speed.kp`、`speed.ki` 与最大俯仰偏置 `max_pitch`，并进行速度阶跃曲线分析。
- **转差环调节**：在线调整 `turn.kp`、`turn.ki` 与最大转向输出 `max`，并进行转差阶跃曲线分析。
- **整车整定 / 视觉**：给出安全的整定顺序，并可调整相机转差给定的更新间隔、加权滤波、变化限幅和失线保持参数。

速度环和转差环页面都会显示目标值、实际值、误差，并在“清空并从 0 s 开始记录”后记录独立实验。每个页面都可以导出当前曲线 CSV；连接期间还会自动把完整遥测保存至 `tool/records/speed_turn_tuning_*.csv`。

平衡角环页同样支持从 0 s 开始记录和 CSV 导出，记录请求/实际俯仰、俯仰误差、角速度、原始平衡输出、左右电机平均得到的实际平衡输出估计及内环饱和时间。整车页还可填写批次、电池电压、地面、轮胎/负载/温度和试验说明；这些信息以及每一条参数命令、控制命令和 ACK 写入同批次的 `speed_turn_tuning_events_*.csv`，从而建立“参数—测试指令—结果”的对应关系。

稳态值为尾部窗口的中位数。噪声用 MAD 推算标准差，调节时间的稳态带取三者最大值：设定的绝对容差、相对容差和 `3σ`。这避免编码器低速量化噪声导致曲线已经稳定却永远判为“未收敛”。

“中位数 + EMA”只用于显示与指标分析，不会改变主板滤波或控制律。优化助手只生成小步 PI 建议，必须由操作者确认、再点击“应用 PI 与限幅”才会下发。

## 分段 PI 的限制

当前主板协议和控制器实现的是固定 `Kp/Ki`，不应由 Wi-Fi 上位机在闭环周期内频繁切换参数：UDP 延迟和丢包会使这种伪分段控制不可靠。工具因此提供两段 PI 的设计提示和细调建议，但不会把它伪装成已启用的分段控制。

如果确实需要运行时分段 PI，应在主板 `VelocityController` / `DifferentialSpeedController` 内实现误差阈值、滞回、积分连续性和遥测回读，并保持 `P,<seq>,...` 这一运输协议不变。

## PWM 与其他性能相关参数

电机 PWM 频率/位宽、左右电机方向、编码器每圈计数与轮径、IMU 轴向和互补滤波、控制周期、倾倒安全角等都在 `include/config/vehicle_config.h` 中初始化。它们会影响平衡、前进和转向，但不是普通运行时调参项：例如重设 PWM 会重新初始化输出，修改编码器/IMU 标定可能使速度或姿态正负方向错误。因此上位机只开放已经由固件提供 ACK 回读的控制器参数和输出限幅；这些硬件/安全参数应修改配置、重新编译烧录，并在车轮悬空时重新验证。
