# 主板 Wi-Fi 平衡调试操作手册

## 1. 安全边界

该功能用于上位机实时查看 IMU、控制状态和串口日志，并在线调整角度环 PID、速度环 PI、平衡点 Trim 与输出限幅。

- 上位机可请求“启动平衡”和“停止平衡”，但启动仍完整复用 `SafetyManager`：自检通过、`STANDBY`、IMU 有效且姿态角安全才会成功，绝不直接使能电机。
- BOOT 按键和串口启停逻辑保持不变。
- 上位机可在 `BALANCING` 状态下下发低速前进给定；不提供无线转向或电机测试命令。`STOP`、IMU 异常和倾倒故障仍优先清零速度并关闭电机。
- 调参时车轮必须悬空；实际启动前将车放在安全地面。网络功能不能替代实体急停。

## 2. 网络与正确地址

| 项目 | 默认值 |
| --- | --- |
| Wi-Fi 名称 | `BALANCE-CAR` |
| 密码 | `balance123` |
| 上位机“主板 IP” | `192.168.4.1` |
| 本地遥测/日志端口 | UDP `9000` |
| 主板命令端口 | UDP `9001` |

`192.168.4.1` 是主板热点网关和命令地址。`192.168.4.255` 是广播地址，只用于未订阅时的遥测回退，不能作为常规主板 IP。

电脑连接 `BALANCE-CAR` 后，在 Windows 终端执行 `ipconfig`，查看 WLAN 适配器 IPv4 地址；它通常为 `192.168.4.2` 或相近地址。运行上位机 `tools/host_pyqt.py`，打开“主板调试”页，填写上述默认值并点击“连接主板”。

上位机会从本地 UDP `9000` 周期发送 `H` 订阅包。订阅成功后，主板串口应显示：

```text
[WIFI] TELEMETRY_SUBSCRIBER=192.168.4.x:9000
[WIFI] TELEMETRY_TX=... TARGET=192.168.4.x:9000 LAST_SEND=OK
```

如果 `TARGET` 始终是 `192.168.4.255:9000`，说明订阅包未被主板接收；确认电脑已连接主板热点、上位机填写 `192.168.4.1`，并重启最新上位机和主板固件。

单无线网卡不能同时连接摄像头热点与主板热点；调试平衡时连接 `BALANCE-CAR`。

## 3. 上位机启停与调参

只有状态为 `STANDBY` 且 IMU 有效时，“启动平衡”按钮才会启用。点击后有二次确认；主板成功回复 `A,<序号>,OK,ARMED`，条件不满足时回复 `ARM_REJECTED`。每次成功启动（包括实体 BOOT 长按）会将速度环给定设为 `0.050 m/s`，而非零速度；停止按钮发送 `STOP`，主板回复 `A,<序号>,OK,STOPPED` 并复用既有 `disarm/reset/clear` 停机流程。

可在线修改的参数：

| 控制环 | 参数 | 安全范围 |
| --- | --- | --- |
| 角度环 | `Kp`、`Ki`、`Kd` | 非负，沿用控制器现有约束 |
| 角度环 | 平衡点 `Trim` | 上位机 `-20°～20°`；对应原有 `setTargetPitchDegrees()` |
| 角度环 | `max_motor` | `0～1`，归一化电机输出上限 |
| 速度环 | `Kp`、`Ki` | 非负，速度环仍为 PI，无 D 项 |
| 速度环 | `max_pitch` | `0～15°`，最大俯仰偏置 |

当前初始参数为 `Trim=-2.24°`、角度环 `Kp=0.15/Ki=0/Kd=0.003`、`max_motor=0.55`，速度环 `Kp=3.0/Ki=0.15/max_pitch=6°`。必须先轮胎悬空确认正速度方向和低速响应；若正速度实际后退，可临时通过串口 `set speed invert 1` 调整 `speed.outputInverted`，不改变 PID 公式。减小限幅通常更安全；增加前确认机械与 PID 已稳定。

在“前进速度给定”区域可设 `0.000–0.250 m/s`。非零给定通过 `0.10 m/s²` 斜坡生效；“回到原地平衡”发送 `0` 并平滑减速。Wi-Fi 断连时保持最后一次有效给定，直到显式归零、`STOP` 或安全故障。

## 4. UDP 协议

主板以约 50 Hz 发送 `T,1,...` 遥测，包含状态、故障码、IMU、速度、电机输出、PID/PI、Trim 和两项输出限幅。上位机只接收并显示，不重新解算 IMU；IMU 数据来自原有 `imuDriver.read()` 与 `attitudeEstimator.update()`。

上位机发送：

```text
H
P,<序号>,balance,kp|ki|kd|trim,<值>
P,<序号>,balance,max_motor,<0–1>
P,<序号>,speed,kp|ki,<值>
P,<序号>,speed,max_pitch,<0–15>
C,<序号>,ARM
C,<序号>,STOP
C,<序号>,DRIVE,<0.000–0.250>
```

`DRIVE` 仅在 `BALANCING` 状态接受；否则回复 `ERR,NOT_BALANCING`。超出范围回复 `ERR,SPEED_RANGE`，格式或非数值速度回复 `ERR,INVALID_SPEED`。主板以 `A,<序号>,OK|ERR,<原因>` 回复命令；以遥测回读的目标/实际速度为实际生效依据。

## 5. 串口日志镜像

上位机“主板串口日志镜像”窗口会显示原主程序的行级串口输出，包括 `[SELFTEST]`、`[STATUS]`、`[TUNING]`、`[CMD]`、`[BALANCE]`、`[OFFLINE]`、`[WIFI]` 和每 500 ms 的 `[TELEMETRY]`。USB 串口输出仍保持不变。

日志使用 `L,<序号>,<原串口文本>` UDP 包发送，窗口保留最近 500 行。主板缓存上电后的最近 32 行，订阅建立后自动补发；日志仅用于调试，不参与平衡控制。

## 6. 排查

| 现象 | 检查项 |
| --- | --- |
| 一直等待遥测 | 必须同时烧录最新主板固件并重启最新上位机；检查串口的 `TELEMETRY_SUBSCRIBER` 与 `TELEMETRY_TX` |
| 有日志、无遥测 | 检查 `T,1,...` 协议与上位机版本是否同步；当前固件已使用真正换行结尾，避免最后一个浮点字段解析失败 |
| 启动按钮不可用 | 等待 `STANDBY`，确认 IMU 有效与遥测正常 |
| `ARM_REJECTED` | 检查自检、IMU 健康度、车辆是否直立及姿态角限制 |
| 两轮转动但实际速度接近 0 | 查看 `[TELEMETRY] wheel_mps_lr`、`tick_delta_lr` 与 `ticks_lr`；前进时左右应同号。当前右编码器已按镜像安装反向。若任一总 tick 不变化，检查该侧 GPIO11/12 或 GPIO13/14 接线、供电和编码器输出。 |
| 没有 ACK | 确认主板地址为 `192.168.4.1`、命令端口为 UDP `9001` |
| UDP 端口被占用 | 上位机会报告本地 UDP `9000` 绑定失败；关闭旧上位机实例或占用端口的程序 |

## 7. 修改热点配置

如附近存在同名车辆或需修改密码，只改 `include/config/wifi_debug_config.h`，然后重新编译烧录。上位机侧 IP/端口必须与配置保持一致。

## 8. 上位机完整使用与测试流程

本节供未参与开发的成员从零使用上位机。开始前确认主板与上位机都来自同一版本，且车轮在调参阶段悬空。

### 8.1 安装与启动

在 `balance_camera` 项目根目录执行：

```powershell
python -m venv venv_host
.\venv_host\Scripts\python.exe -m pip install -r tools\requirements.txt
.\venv_host\Scripts\python.exe tools\host_pyqt.py
```

已有 `venv_host` 时只需最后一行。依赖为 PyQt5、OpenCV、NumPy、Requests 与 PySerial。主板需烧录本项目对应的最新固件后再上电；串口看到 `[WIFI] AP=READY` 才表示热点已启动。

### 8.2 建立通信

1. 在电脑 Wi-Fi 列表连接 `BALANCE-CAR`，密码为 `balance123`。
2. 打开上位机的“主板调试”页。
3. 填写主板 IP `192.168.4.1`、本地遥测端口 `9000`、主板命令端口 `9001`；不要把 `.255` 填入主板 IP。
4. 点击“连接主板”。
5. 成功标准：连接状态变为“已收到主板遥测”，显示包序号、包龄和频率；主板串口日志镜像出现 `[WIFI] H rx=...` 与 `[WIFI] TX ok=... sub=1`。

如果连接后仍等待遥测，先不要调参。检查上位机日志是否有“已收到首个主板遥测包”；再检查主板日志中的 `target=192.168.4.x:9000` 是否等于电脑 WLAN IPv4 地址和端口 9000。

### 8.3 阅读界面

- “实时状态”：安全状态、故障、IMU 有效标志、俯仰角、角速度、三轴加速度/陀螺仪、速度、电机输出和 UDP 包龄。
- “在线 PID / PI 参数”：当前输入框会由主板遥测回读更新；输入框获得焦点时不会被遥测覆盖。
- “主板串口日志镜像”：显示原主程序的调试行与 Wi-Fi 诊断，保留最近 500 行。当前诊断固件暂时关闭了高频 `[TELEMETRY]` 串口行，以便突出 Wi-Fi 链路；这不影响结构化 IMU 遥测和控制。

### 8.4 参数下发测试

1. 保持车轮悬空，先记录界面回读的默认参数。
2. 只把一个参数改一个很小的量，例如平衡点 Trim 从 `-0.80` 改为 `-0.75`。
3. 点击“应用全部参数”。该按钮会发送全部 8 项：角度 Kp/Ki/Kd、Trim、max_motor、速度 Kp/Ki、max_pitch。
4. 成功标准：日志依次出现 `A,<序号>,OK,APPLIED`；约 1 秒内输入框回读为主板实际值。
5. 若出现 `INVALID_PARAMETER`，确认上位机和主板均已更新到最新版本后重启；不要继续尝试启动平衡。

### 8.5 启动与停止测试

1. 停止调参，将车辆放到安全地面、扶正并清空周围区域。
2. 等待实时状态显示 `STANDBY` 且 IMU 为“有效”。此时“启动平衡”按钮才会启用。
3. 点击“启动平衡”，阅读确认框后确认。成功时收到 `A,<序号>,OK,ARMED`，状态变为 `BALANCING`。
4. 启动后确认“目标 / 实际速度”为约 `0.050 / 0.000 m/s` 起步；先在轮胎悬空状态确认方向，再进行地面低风险测试。
5. 状态为 `BALANCING` 时，在“前进速度给定”输入 `0.050 m/s` 后点击“设置前进速度”；成功 ACK 为 `DRIVE_ACCEPTED`。再试 `0.100 m/s`，观察目标速度平滑变化、实际速度同向跟随，俯角输出不超过 `±6°`、电机输出不超过 `±0.55`。
6. 点击“回到原地平衡”，确认目标速度平滑回到 0；任意时刻点击红色“停止平衡”，应收到 `A,<序号>,OK,STOPPED`，状态回到 `STANDBY` 或保留故障状态。
7. 若为 `ARM_REJECTED`，检查自检是否通过、IMU 是否健康、车身是否直立；若 `NOT_BALANCING`，先完成启动，不要反复下发速度命令。

### 8.6 建议验收记录

每次测试记录固件版本、Wi-Fi 名称、主板/电脑 IP、默认 PID/PI/Trim/限幅、`H rx` 与 `TX ok/fail` 日志、启动/停止 ACK 以及异常现象。这样另一位成员可复现问题，而无需连接 USB 串口。
