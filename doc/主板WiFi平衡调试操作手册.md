# 主板 Wi-Fi 平衡调试操作手册

## 1. 安全边界

该功能用于上位机实时查看 IMU、控制状态和串口日志，并在线调整角度环 PID、速度环 PI、平衡点 Trim 与输出限幅。

- 上位机可请求“启动平衡”和“停止平衡”，但启动仍完整复用 `SafetyManager`：自检通过、`STANDBY`、IMU 有效且姿态角安全才会成功，绝不直接使能电机。
- BOOT 按键和串口启停逻辑保持不变。
- 不提供无线目标速度、转向或电机测试命令。
- 调参时车轮必须悬空；实际启动前将车放在安全地面。网络功能不能替代实体急停。

## 2. 网络与正确地址

| 项目 | 默认值 |
| --- | --- |
| Wi-Fi 名称 | `BALANCE-CAR` |
| 密码 | `balance123` |
| 上位机“主板 IP” | `192.168.4.1` |
| 本地遥测/日志端口 | UDP `9000` |
| 主板命令端口 | UDP `9001` |

`192.168.4.1` 是主板热点网关和命令地址。`192.168.4.255` 是广播地址，只用于未订阅时的遥测回退，不能作为常规主板 IP。

电脑连接 `BALANCE-CAR` 后，在 Windows 终端执行 `ipconfig`，查看 WLAN 适配器 IPv4 地址；它通常为 `192.168.4.2` 或相近地址。运行上位机 `tools/host_pyqt.py`，打开“主板调试”页，填写上述默认值并点击“连接主板”。

上位机会从本地 UDP `9000` 周期发送 `H` 订阅包。订阅成功后，主板串口应显示：

```text
[WIFI] TELEMETRY_SUBSCRIBER=192.168.4.x:9000
[WIFI] TELEMETRY_TX=... TARGET=192.168.4.x:9000 LAST_SEND=OK
```

如果 `TARGET` 始终是 `192.168.4.255:9000`，说明订阅包未被主板接收；确认电脑已连接主板热点、上位机填写 `192.168.4.1`，并重启最新上位机和主板固件。

单无线网卡不能同时连接摄像头热点与主板热点；调试平衡时连接 `BALANCE-CAR`。

## 3. 上位机启停与调参

只有状态为 `STANDBY` 且 IMU 有效时，“启动平衡”按钮才会启用。点击后有二次确认；主板成功回复 `A,<序号>,OK,ARMED`，条件不满足时回复 `ARM_REJECTED`。停止按钮发送 `STOP`，主板回复 `A,<序号>,OK,STOPPED` 并复用既有 `disarm/reset/clear` 停机流程。

可在线修改的参数：

| 控制环 | 参数 | 安全范围 |
| --- | --- | --- |
| 角度环 | `Kp`、`Ki`、`Kd` | 非负，沿用控制器现有约束 |
| 角度环 | 平衡点 `Trim` | 上位机 `-20°～20°`；对应原有 `setTargetPitchDegrees()` |
| 角度环 | `max_motor` | `0～1`，归一化电机输出上限 |
| 速度环 | `Kp`、`Ki` | 非负，速度环仍为 PI，无 D 项 |
| 速度环 | `max_pitch` | `0～15°`，最大俯仰偏置 |

建议先调平衡点 Trim 与角度环 PD，再少量引入角度 Ki，最后调速度环 PI。输出限幅从保守值开始：默认 `max_motor=0.35`、`max_pitch=6°`。减小限幅通常更安全；增加前确认机械与 PID 已稳定。

## 4. UDP 协议

主板以约 50 Hz 发送 `T,1,...` 遥测，包含状态、故障码、IMU、速度、电机输出、PID/PI、Trim 和两项输出限幅。上位机只接收并显示，不重新解算 IMU；IMU 数据来自原有 `imuDriver.read()` 与 `attitudeEstimator.update()`。

上位机发送：

```text
H
P,<序号>,balance,kp|ki|kd|trim,<值>
P,<序号>,balance,max_motor,<0–1>
P,<序号>,speed,kp|ki,<值>
P,<序号>,speed,max_pitch,<0–15>
C,<序号>,ARM
C,<序号>,STOP
```

主板以 `A,<序号>,OK|ERR,<原因>` 回复命令。以遥测回读值为实际生效值。

## 5. 串口日志镜像

上位机“主板串口日志镜像”窗口会显示原主程序的行级串口输出，包括 `[SELFTEST]`、`[STATUS]`、`[TUNING]`、`[CMD]`、`[BALANCE]`、`[OFFLINE]`、`[WIFI]` 和每 500 ms 的 `[TELEMETRY]`。USB 串口输出仍保持不变。

日志使用 `L,<序号>,<原串口文本>` UDP 包发送，窗口保留最近 500 行。主板缓存上电后的最近 32 行，订阅建立后自动补发；日志仅用于调试，不参与平衡控制。

## 6. 排查

| 现象 | 检查项 |
| --- | --- |
| 一直等待遥测 | 必须同时烧录最新主板固件并重启最新上位机；检查串口的 `TELEMETRY_SUBSCRIBER` 与 `TELEMETRY_TX` |
| 有日志、无遥测 | 检查 `T,1,...` 协议与上位机版本是否同步；当前固件已使用真正换行结尾，避免最后一个浮点字段解析失败 |
| 启动按钮不可用 | 等待 `STANDBY`，确认 IMU 有效与遥测正常 |
| `ARM_REJECTED` | 检查自检、IMU 健康度、车辆是否直立及姿态角限制 |
| 没有 ACK | 确认主板地址为 `192.168.4.1`、命令端口为 UDP `9001` |
| UDP 端口被占用 | 上位机会报告本地 UDP `9000` 绑定失败；关闭旧上位机实例或占用端口的程序 |

## 7. 修改热点配置

如附近存在同名车辆或需修改密码，只改 `include/config/wifi_debug_config.h`，然后重新编译烧录。上位机侧 IP/端口必须与配置保持一致。
