# 主板 Wi-Fi 平衡调试操作手册

## 1. 用途与安全边界

本功能用于在上位机实时观察主板 IMU 与控制状态，并在线调整角度环 PID、速度环 PI 参数。

- 不提供无线启动、急停、目标速度或转向命令。
- 平衡启动/停止仍使用现有 BOOT 按键和串口命令。
- 调参前必须让车轮悬空或置于可靠支架；网络调试不能替代实体急停。

## 2. 连接方法

烧录主板程序并上电后，串口应输出：

```text
[WIFI] AP=READY SSID=BALANCE-CAR IP=192.168.4.1 TELEMETRY_PORT=9000 COMMAND_PORT=9001
```

电脑连接热点：

| 项目 | 默认值 |
| --- | --- |
| Wi-Fi 名称 | `BALANCE-CAR` |
| 密码 | `balance123` |
| 主板 IP | `192.168.4.1` |
| 本地遥测 UDP 端口 | `9000` |
| 主板命令 UDP 端口 | `9001` |

运行 `tools/host_pyqt.py`，打开“主板调试”页，保持默认 IP 和端口，点击“连接主板”。收到遥测后，状态会显示为绿色，包龄通常低于 100 ms，接收频率约为 50 Hz。

相机热点与主板热点都使用独立 AP；只有一个无线网卡时，调试平衡请连接 `BALANCE-CAR`，不能同时接收摄像头图传。

## 3. 数据与调参协议

主板每 20 ms 在 UDP `9000` 广播一条 `T,1,...` ASCII 遥测。它包含序号、主板时间、安全状态、故障码、IMU、速度环、电机输出和当前生效的六个调参值。

上位机仅发送：

```text
P,<序号>,balance,kp|ki|kd|trim,<值>
P,<序号>,speed,kp|ki,<值>
```

主板回复 `A,<序号>,OK,APPLIED` 或错误原因。遥测中的参数值才是最终生效值；上位机会自动回读刷新输入框。

## 4. 推荐调参流程

1. 车轮悬空，确认 IMU 有效、静止时俯仰角稳定，先处理机械重心和安装松动问题。
2. 保持速度环 `Kp/Ki` 为默认值或零，先调整角度环 `Kp` 与 `Kd`，使车辆姿态响应快速但不高频振荡。
3. 用 `trim` 消除静止时持续向一侧滚动的机械零点偏差；每次改动应很小。
4. 仅在 PD 与机械零点稳定后，少量增加角度 `Ki`，避免积分累积造成突然冲击。
5. 最后调整速度环 PI：先 `Kp`，再极小量 `Ki`。当前速度环是 PI，不含 D 项。
6. 每次只改变一个参数并观察俯仰角、角速度和电机输出；出现持续增幅振荡立即通过实体按键/串口停止，不要依赖网络。

## 5. 排查

| 现象 | 检查项 |
| --- | --- |
| 无遥测 | 电脑是否连接 `BALANCE-CAR`、串口是否有 `[WIFI] AP=READY`、端口是否为 9000 |
| 包龄持续增长 | 检查热点信号、电脑防火墙 UDP 入站规则，以及是否误连摄像头热点 |
| 参数无 ACK | 主板命令端口应为 9001；确认命令仅使用六项允许参数 |
| ACK 成功但数值异常 | 查看遥测回读值；负 PID 会被现有控制器限制为 0，trim 允许为负 |
| 主板 AP 未启动 | 检查串口的 `[WIFI] AP_START=FAILED` 或 `[WIFI] UDP_BIND=FAILED`，再重启主板 |

## 6. 修改热点配置

如附近有同名车辆或需修改密码，只改主板项目的 `include/config/wifi_debug_config.h`，然后重新编译烧录。上位机 IP/端口与该配置保持一致。
